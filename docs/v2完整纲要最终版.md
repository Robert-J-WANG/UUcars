```typescript
// useDebounce实现
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value)
  useEffect(() => {
    const timer = setTimeout(() => setDebouncedValue(value), delay)
    return () => clearTimeout(timer)
  }, [value, delay])
  return debouncedValue
}
```

## Step 58 · Refresh Token机制

**产出：** Access Token过期后自动续期，用户不需要重新登录

后端新增：

```
POST /auth/refresh   用Refresh Token换新Access Token
POST /auth/logout    撤销Refresh Token
```

后端数据库Migration（新增RefreshTokens表）：

```
Id          int, PK
UserId      int, FK → Users.Id
Token       nvarchar(200), NOT NULL, UNIQUE
ExpiresAt   datetime2, NOT NULL
IsRevoked   bit, DEFAULT 0
CreatedAt   datetime2
```

知识点：

- Access Token（60分钟短期）vs Refresh Token（30天长期）的设计原因
- HttpOnly Cookie存储Refresh Token（前端JS无法读取，防XSS）
- Token轮换策略（每次刷新生成新Refresh Token，旧的作废）
- Axios响应拦截器实现自动刷新：
    - 401时自动调用 `/auth/refresh`
    - 刷新成功则重试原请求
    - 刷新失败则跳转登录页
- 并发请求时的刷新队列（防止多个401同时触发多次刷新）
