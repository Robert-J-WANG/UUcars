#### Step 38 · 邮箱验证注册

V1 注册时 `EmailConfirmed = false`，这一步让它真正工作：

```
POST /auth/register          → 创建账号 + 发送验证邮件
GET  /auth/verify-email?token=xxx  → 验证邮箱，EmailConfirmed = true
POST /auth/resend-verification    → 重新发送验证邮件
```

数据库 Migration（新增字段到 Users 表）：

```
EmailConfirmationToken         nvarchar(200), NULL
EmailConfirmationTokenExpiry   datetime2, NULL
```

知识点：

- EF Core 新增字段的 Migration 流程（和 V1 初始 Migration 的区别）
- Token 生成（`Convert.ToBase64String(RandomNumberGenerator.GetBytes(32))`）
- Token 过期时间设计（24小时）
- 已验证邮箱重复请求的处理

#### Step 42 · Refresh Token 机制

```
POST /auth/refresh   → 用 Refresh Token 换新 Access Token
POST /auth/logout    → 撤销 Refresh Token
```

数据库 Migration（新增 RefreshTokens 表）：

```
Id          int, PK
UserId      int, FK → Users.Id
Token       nvarchar(200), NOT NULL, UNIQUE
ExpiresAt   datetime2, NOT NULL
IsRevoked   bit, DEFAULT 0
CreatedAt   datetime2
```

知识点：

- Access Token（60分钟）vs Refresh Token（30天）
- Refresh Token 存 HttpOnly Cookie（前端 JS 无法读取，防 XSS）
- Token 轮换（每次刷新生成新 Refresh Token，旧的作废）
- 前端如何自动刷新（Axios 响应拦截器，401 时自动重试）——这部分在 Step 47 前端实现

这是把项目真正发布到互联网的关键一步。

**方案选择：DigitalOcean Droplet + Nginx + Let's Encrypt**

理由：

- 最便宜的 VPS（$6/月），真实生产环境
- 教会最底层的部署流程（不是 Heroku / Vercel 那种黑盒）
- 知识可迁移到任何云服务商

**知识点：**

服务器初始化：

```
购买 DigitalOcean Droplet（Ubuntu 22.04）
配置 SSH 密钥登录（禁用密码登录）
安装 Docker + docker-compose
配置防火墙（UFW：只开 22/80/443 端口）
```

域名与 HTTPS：

```
域名 DNS 解析到服务器 IP（A 记录）
申请 Let's Encrypt 免费 SSL 证书（Certbot 自动申请 + 续期）
Nginx 配置 HTTPS（HTTP 自动跳转 HTTPS）
```

部署流程：

```
GitHub Actions 构建镜像推送到 ghcr.io
服务器 pull 镜像 + docker compose up
后续更新：一键 deploy 脚本
```

更新 GitHub Actions（`ci.yml`）：

```
push 到 main → 构建镜像 → SSH 到服务器 → 拉取新镜像 → 重启容器
```

知识点：

- SSH 免密登录服务器（公钥认证）
- Nginx 反向代理配置（`proxy_pass` 到 API 容器）
- Let's Encrypt + Certbot（自动申请和续期 SSL）
- `docker compose pull + up -d`（零停机更新）
- 环境变量在服务器上的管理（`.env` 文件，不进 Git）
- 日志查看（`docker compose logs -f`）
